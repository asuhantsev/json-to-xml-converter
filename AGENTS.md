# AGENTS.md

## Назначение
Агент ведет проект `json-to-xml-tgchat` к стабильному, чистому и поддерживаемому состоянию:
- исправляет баги и регрессии;
- упрощает код без изменения пользовательского поведения без необходимости;
- поддерживает прозрачный трекинг работ и статусов.

## Главные правила
- Делать небольшие, проверяемые изменения.
- Перед правками читать контекст и затронутые участки кода.
- Не ломать основной сценарий GUI-конвертации Telegram JSON -> XML.
- После изменений запускать минимальную проверку (smoke).

## Трекинг задач: только Beads (`bd`)
В этом проекте задачи, эпики и подзадачи ведутся только через `beads`.

Не использовать:
- markdown TODO-листы;
- отдельные sqlite/csv/json-таск-трекеры;
- дублирование задач в нескольких местах.

Использовать:
- `bd ready` — что можно брать в работу;
- `bd create` — создание эпиков/задач;
- `bd update` — смена статуса/приоритета/исполнителя;
- `bd close` — закрытие выполненной задачи;
- `bd list` / `bd show <id>` — просмотр бэклога и деталей.

## Минимальный рабочий цикл
1. Взять готовую задачу: `bd ready`.
2. Уточнить критерий приемки.
3. Внести минимальный рабочий фикс.
4. Проверить результат локально.
5. Обновить задачу в `bd` (заметки, статус, закрытие).

## Ограничения
- Не применять разрушительные git-команды.
- Не удалять пользовательские экспортированные данные.
- Не делать несвязанный рефакторинг вне активной задачи.
